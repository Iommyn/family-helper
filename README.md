# Family Helper — трекер домашних дел

приложение для распределения и контроля домашних обязанностей между членами
семьи. Написано на чистом JavaScript без фреймворков, данные хранятся
локально в браузере (`localStorage`)

## Что может

**Задачи**: создание, редактирование, удаление; категории (уборка, покупки,
  ремонт, прочее и свои собственные); срок выполнения.
**Исполнитли**: добавление членов семьи, назначение задачи на одного или
  нескольких человек.
**Контроль выполнения**: отметка «выполнено» / возврат в работу, история
  выполненных дел, напоминания о приближающихся и просроченных сроках.
**Аналитика**: подсчёт выполненных задач по каждому участнику, рейтинг
  активности, две столбчатые диаграммы (по участникам и по категориям).
**Поиск и фильтры** по статусу, категории, исполнителю, тексту; сортировка по
  сроку, названию и дате создания
**Дополнительно**: экспорт всех данных в JSON, добавление своих категорий,
  адаптивная вёрстка (корректно от 320px), сброс данных.

## Как запустить сайт

Сборка не нужна. Достаточно открыть файл `index.html` в браузере

Вариант 1 — двойной клик по `index.html`

Вариант 2 — через локальный сервер:

```bash
cd family-helper
python3 -m http.server 8000
# затем открыть http://localhost:8000
```

## Тесты

Откройте `tests/tests.html` в браузере — на странице появится отчёт о
пройденных проверках (участники, CRUD задач, назначение, выполнение, рейтинг,
уведомления, сохранение в localStorage)

## Структура проекта

```
family-helper/
├── index.html              # html файл, нужен для разметки
├── styles/
│   └── style.css           # стили для рtml файла
├── js/
│   ├── storage.js          # в качестве хранения localStorage, чтение и запись
│   ├── users.js            # члены семьи
│   ├── tasks.js            # задачи CRUD, фильтры, сортировка, статусы
│   ├── analytics.js        # рейтинг и статистика
│   ├── notifications.js    # напоминания о срока
│   ├── charts.js           # диаграммы на canvas
│   ├── ui.js               # отрисовка dom дерева
│   └── app.js              # контроллер состояние и обработчики событий
├── tests/
│   ├── tests.html
│   └── tests.js
└── README.md
```

Разделение ответственности, модули данных (`storage`, `users`, `tasks`,
`analytics`, `notifications`) не знают про dom, модуль `ui` отвечает только за
отрисовку, `app` связывает их и обрабатывает события пользователя

## Модель данных

**Task**

| Поле          | Тип      | Описание                                   |
|---------------|----------|--------------------------------------------|
| `id`          | string   | уникальный идентификатор                   |
| `title`       | string   | название (обязательно)                     |
| `description` | string   | описание                                   |
| `category`    | string   | категория из списка                        |
| `dueDate`     | string   | срок в формате `YYYY-MM-DD` или пустая строка |
| `assignees`   | string[] | id назначенных исполнителей                |
| `status`      | string   | `planned` или `done`                       |
| `createdAt`   | string   | дата создания                              |
| `completedAt` | string   | дата выполнения                            |
